### مقدمة: ما هو الـ Git ولماذا نستخدمه؟
تخيل أنك تكتب كتاباً، وتخشى أن تخطئ فتنسخ الملف كل يوم (كتاب_نسخة1، كتاب_نسخة2... إلخ). 
**Git** هو نظام حفظ ذكي (Version Control System). بدلاً من نسخ الملفات، يلتقط "لقطات" (Snapshots) لكل تعديل تقوم به، ويسمح لك بالعودة لأي لقطة في أي وقت، كما يسمح لفريق كامل بالعمل على نفس المشروع في نفس الوقت دون أن يمسح أحد عمل الآخر.

*ملاحظة: Git هو البرنامج الموجود على جهازك، بينما GitHub هو موقع الويب الذي ترفع عليه مشروعك لمشاركته مع العالم.*

### المرحلة الأولى: الإعداد والبداية (الصفر)

**1. تثبيت Git:**
*   **Windows:** حمله من موقع git-scm.com ونصّبه.
*   **Mac:** افتح الـ Terminal واكتب `git` (سيطلب منك تثبيته تلقائياً).
*   **Linux:** `sudo apt install git`

**2. الإعداد الأولي (مرة واحدة فقط):**
بعد فتح الـ Terminal (أو Git Bash في ويندوز)، عرّف بنفسك لـ Git ليضع اسمك على تعديلاتك:
```bash
git config --global user.name "اسمك هنا"
git config --global user.email "بريدك_الالكتروني@هنا.com"
```

**3. بدء مشروع (المربع السحري):**
افتح مجلداً جديداً لمشروعك، واكتب الأمر التالي لتحويل هذا المجلد العادي إلى مجلد يراقبه Git:
```bash
git init
```

### المرحلة الثانية: المثلث الذهبي (أساس العمل)

لفهم Git، يجب أن تتخيز مثلثاً من 3 مناطق:
1. **Working Directory (مساحة العمل):** الملفات التي تعدلها أنت الآن.
2. **Staging Area (منطقة الانتظار):** كعربة التسوق، تضع فيها الملفات المعدلة استعداداً لحفظها.
3. **Local Repository (المستودع المحلي):** قاعدة البيانات التي تُحفظ فيها اللقطات (Snapshots).

**الأوامر الأساسية:**
*   لمعرفة حالة ملفاتك (ماذا تعدل؟):
    ```bash
    git status
    ```
*   لوضع ملف في عربة التسوق (Staging):
    ```bash
    git add filename.txt
    ```
*   لوضع كل الملفات المعدلة في عربة التسوق دفعة واحدة:
    ```bash
    git add .
    ```
*   لحفظ التغييرات بشكل رسمي في المستودع المحلي (التقاط لقطة):
    ```bash
    git commit -m "وصف مختصر لما قمت به"
    ```

### المرحلة الثالثة: السفر عبر الزمن (السجلات والرجوع)

*   لمعرفة كل اللقطات (Commits) التي قمت بها:
    ```bash
    git log
    ```
    *(للخروج من الشاشة اضغط حرف `q`)*

*   **كيف تتراجع عن تعديل لم تعجبك؟**
    إذا كنت تعدل في ملف ولم تقم بـ `git add` بعد، وتريد إرجاعه لآخر نسخة محفوظة:
    ```bash
    git restore filename.txt
    ```

### المرحلة الرابعة: الفروع (Branches) - قلب الاحتراف

تخيل أن لعبة الكتاب الرئيسية تسمى `main`. لا تريد العبث فيها وأنت تجرب فكرة جديدة. لذلك تصنع "عالياً موازياً" (فرعاً جديداً).

*   لإنشاء فرع جديد والانتقال إليه فوراً:
    ```bash
    git checkout -b new-feature
    ```
    *(أو الأمر الأحدث: `git switch -c new-feature`)*

*   للعودة للفرع الرئيسي:
    ```bash
    git checkout main
    ```

*   **دمج الفروع (Merge):**
    عندما تنتهي من الميزة الجديدة في الفرع `new-feature` وتريد ضمها للمشروع الرئيسي `main`:
    1. ارجع للفرع الرئيسي: `git checkout main`
    2. ادمج: `git merge new-feature`

### المرحلة الخامسة: رفع المشروع للسحابة (GitHub)

الآن مشروعك محفوظ على جهازك فقط. لنرفعه للإنترنت (GitHub/GitLab).

**1. اربط جهازك بالمستودع البعيد:**
بعد إنشاء مستودع جديد على GitHub، سيعطيك رابطاً. اربطه بجهازك:
```bash
git remote add origin https://github.com/username/repo-name.git
```

**2. رفع الكود (Push):**
لرفع تعديلاتك من جهازك إلى الإنترنت:
```bash
git push -u origin main
```
*(في المرات القادمة ستكتفي بكتابة `git push` فقط).*

**3. سحب الكود (Pull):**
إذا كنت تعمل في فريق، وزميلك رفع تعديلات على GitHub، يجب أن تحدث جهازك بها:
```bash
git pull
```

### المرحلة السادسة: مستوى المحترفين (Advanced Commands)

هنا تصبح خبيراً:

**1. التراجع عن Commit (إذا ندمت على حفظ قمت به):**
*   لتراجع مع الاحتفاظ بالتعديلات في ملفاتك:
    ```bash
    git reset --soft HEAD~1
    ```
*   لتراجع ومسح التعديلات نهائياً (احذر!):
    ```bash
    git reset --hard HEAD~1
    ```

**2. تعديل آخر Commit (نسيت إضافة ملف أو تريد تغيير الرسالة):**
```bash
git commit --amend -m "الرسالة الجديدة"
```

**3. إخفاء التعديلات (Stash):**
أنت تعمل على ميزة، وفجأة طلب منك مديرك إصلاح خطأ عاجل في المشروع الرئيسي. لا تريد أن تفقد عملك الحالي ولا تريد عمل commit غير مكتمل. خبئه!
```bash
git stash
```
بعد أن تنتهي من الإصلاح العاجل وتعود لفرعك، استرجع عملك المخبأ:
```bash
git stash pop
```

**4. Rebase (إعادة التأسيس) - أداة المحترفين لتنظيف التاريخ:**
بدلاً من دمج الفروع (Merge) الذي يترك تاريخاً متشعباً (شجرياً)، الـ Rebase يأخذ تعديلاتك ويلحقها بنهاية الفرع الرئيسي ليصبح التاريخ خطاً مستقيماً نظيفاً.
```bash
git checkout new-feature
git rebase main
```
*(قاعدة ذهبية: لا تستخدم Rebase على فروع مشتركة مع الآخرين، استخدمه فقط على فروعك المحلية).*

**5. Cherry-Pick (قطف الكرز):**
تريد تعديلاً واحداً (Commit واحد) من فرع آخر دون دمج الفرع كله؟
```bash
git cherry-pick <commit-hash>
```

**6. آلة الزمن السحرية (Reflog):**
إذا قمت بـ `reset --hard` بالخطأ ومسحت كل شيء، لا تَفْزَع! Git يحفظ كل حركة تقوم بها لفترة.
```bash
git reflog
```
سيعرض لك كل تحركاتك، انسخ الـ hash الخاص بالنقطة التي تريد العودة إليها، ثم:
```bash
git reset --hard <hash>
```

### نصائح الخبراء الذهبية:
1. **Commit صغير ومتكرر:** لا تترك عمل اليوم كله وتعمل Commit واحد في النهاية. اعمل Commits صغيرة لكل ميزة أو إصلاح.
2. **رسائل واضحة:** بدلاً من `update`، اكتب `fix: resolve login button alignment issue`.
3. **ملف .gitignore:** لا ترفع ملفات النظام أو السرية (مثل كلمات المرور أو ملفات `node_modules`). أنشئ ملفاً باسم `.gitignore` واكتب فيه أسماء الملفات التي تريد من Git تجاهلها.

### خلاصة الرحلة:
`git init` -> `git add .` -> `git commit -m "msg"` -> `git push`
